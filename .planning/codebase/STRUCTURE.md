## Directory Purposes

**`electron/`:**
- Purpose: Electron app (window management, IPC, file dialogs, crypto storage)
- Contains: Main process, preload bridge, HTML/CSS/JS UI
- Key files: `main.js` (271 lines), `index.html` (2122 lines, single self-contained file with all CSS/JS inline)

**`python/`:**
- Purpose: Flask REST API backend
- Contains: Document parsing, solicitation extraction, quote generation
- Key files: `server.py` (637 lines, includes all route handlers and helpers), `requirements.txt` (pip deps)

**`scripts/`:**
- Purpose: Build helpers and platform-specific installers
- Contains: macOS post-install script, build notes
- Key files: `Install Helper.command` (runs after DMG install to set quarantine flags)

**`resources/`:**
- Purpose: Platform-specific metadata
- Contains: macOS code signing entitlements
- Key files: `entitlements.mac.plist`

**`testdata/`:**
- Purpose: Development fixtures and testing
- Contains: Sample vendor data, quote schema, test runner
- Key files: `quote_input.json` (13 line items with size/qty/price), `run.py` (local test)

**`dist/`:**
- Purpose: Built Electron application (generated)
- Contains: Platform-specific installers and app bundles
- Committed: No (generated by `npm run build:win` / `npm run build:mac`)

**`dist-backend/`:**
- Purpose: PyInstaller-compiled backend binary (generated)
- Contains: Standalone Python executable (solicitationquoter-backend.exe on Windows, solicitationquoter-backend on macOS)
- Committed: No (generated by `pyinstaller` command)

## Key File Locations

**Entry Points:**
- `package.json`: `"main": "electron/main.js"` — app entry point
- `electron/main.js`: Spawns backend, creates window, loads `electron/index.html` after `/ping` succeeds
- `python/server.py`: Flask app started by main process or PyInstaller binary

**Configuration:**
- `electron-builder.json`: Platform targets, DMG/NSIS config, resource embedding
- `solicitationquoter-backend.spec`: PyInstaller spec (bundles python/server.py into standalone binary)
- `package.json`: Scripts (`npm start`, `npm run build:win`, `npm run build:mac`)

**Core Logic:**
- `python/server.py`: All backend logic (extract, ai_extract, extract_line_items, generate_quote)
- `electron/index.html`: All UI logic (wizard steps, form validation, fetch calls)

**Testing:**
- `testdata/quote_input.json`: Example data for manual testing of quote generation
- `testdata/run.py`: Local test runner (uses testdata JSON files to call backend)

## Naming Conventions

**Files:**
- HTML: lowercase + `.html` (e.g., `index.html`, `loading.html`)
- Python: lowercase + `_` for multi-word (e.g., `server.py`)
- JavaScript: lowercase + `.js` (e.g., `main.js`, `preload.js`)
- JSON: lowercase + `.json` (e.g., `package.json`, `quote_input.json`)

**Directories:**
- Top-level: lowercase, no separators (e.g., `electron`, `python`, `scripts`)
- Generated: lowercase, dash-separated (e.g., `dist`, `dist-backend`)

**Functions (Python):**
- snake_case (e.g., `parse_pdf()`, `extract_data()`, `generate_quote()`)

**Functions (JavaScript):**
- camelCase in renderer (embedded in HTML); constructors PascalCase (e.g., `createWindow()`)

**Variables:**
- Python: snake_case (e.g., `api_key`, `tmp_path`, `line_items`)
- JavaScript: camelCase (e.g., `currentStep`, `solicitationData`)
- Constants: SCREAMING_SNAKE_CASE (e.g., `PORT = 5199`, `MAX_LOGO`)

## Where to Add New Code

**New Feature (solicitation parsing enhancement):**
- Primary code: `python/server.py` (add function under "PARSERS" or "RULE-BASED EXTRACTOR" section)
- Tests: `testdata/run.py` (add test case)
- Example: New `extract_foo()` function for custom field extraction

**New UI Step or Component:**
- Implementation: `electron/index.html` inside `<script>` tag (all UI is one file)
- Style: Add to existing `<style>` tag (CSS variables available at `:root`)
- Logic: Add event listener and state update in JavaScript section
- Example: New form section for additional vendor fields

**New Utility/Helper:**
- Shared parsing helpers: `python/server.py` (above route definitions)
- Shared UI helpers: `electron/index.html` `<script>` section

**New API Endpoint:**
- Location: `python/server.py` (add route with `@app.route()` decorator)
- Pattern: Return `jsonify()` for JSON responses; handle errors in `try/except` with `traceback.print_exc()`
- CORS: Already configured in `@app.after_request` decorator

**New File Dialog:**
- Location: `electron/main.js` (add `ipcMain.handle()`)
- Expose via: `electron/preload.js` (add to `contextBridge.exposeInMainWorld()`)
- Call from renderer: `window.api.newDialogName()` in `electron/index.html`
